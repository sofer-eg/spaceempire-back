# Radar — visibility-limit mechanic (phase 10.20)

Спека механики радара при портировании StarWind → spaceempire. Сопровождает
`starwind/docs/tasks/phase10-20-radar-visibility.md`. Ведётся послойно
L1→L5; этот документ покрывает L1–L4 (L5 спутники блокируется 10.15).

## Что делает оригинал StarWind

`CMapRadar` (`starwind/includes/low_level_classes.php:95-713`) формирует
видимый список вокруг корабля игрока:

- **Малый радар `RadarRange`** — корабли, дроны, ракеты, астероиды, контейнеры.
- **Большой радар `RadarRangeBig = RadarRange × 5`** — крупные объекты
  (станции, верфи, ТС, ворота) видны в ~5× дальше.
- База — `ships.radar` (`db.sql:2368`), плоское `default 640` (`db.sql:9063`),
  **не зависит от класса** (в `ct_ship_classes` колонки `radar` нет).
- `up_scanner` расширяет эффективную дальность; `up_hide` ставит
  `object_hidden` и прячет корабль от чужих радаров; спутники раскрывают
  сектор.

## Калибровка координат (зафиксировано)

Старый радар 640 задан в **старой** системе координат (выход ворот ≈ ±1280
на сторону). В spaceempire сектор — круг `BoundsRadius = 5000` (задача 3.10),
до 10.20 действовал плоский AOI = 5000 (виден весь сектор). Дословно переносить
640 нельзя (новое пространство много крупнее относительно объектов), поэтому
радар **заякорён к доле сектора**, а не к старому числу.

**Выбранные значения (world-units, сектор radius 5000):**

| Категория | Малый радар | Роль |
|-----------|-------------|------|
| M5 Scout      | 3500 | разведчик — видит дальше всех |
| M6 Corvette   | 3000 | корвет |
| M3/M4 Fighter | 2800 | истребители |
| TL Freighter  | 2600 | супертранспорт |
| M1/M2 Capital | 2400 | носитель/эсминец (свой радар скромнее, но их самих видно издалека) |
| TS Transport  | 2200 | транспорт |
| XX / default  | 2800 | спец / неизвестный класс |

- Все < 5000 — видимость **реально ограничена** (до 10.20 был весь сектор).
- **Большой радар (L2):** `RadarRange × K`, `K = cfg.RadarBigMultiplier`
  (дефолт **2.5**, откалибровано вниз с оригинальных ×5). При малом ~2800
  большой ~7000 — крупные объекты дальше малых, но **меньше диаметра сектора
  (до 10000)**, так что статика входит/выходит дельтами по мере движения, а не
  видна вся сразу.
- **Fallback:** корабль без класса (`RadarRange <= 0`, легаси/скафандр) →
  `cfg.AOIRadius` (5000), прежнее плоское поведение.

Источник радара — каталог классов: `balance.ShipClass.Radar`. В YAML поля нет
(в `ct_ship_classes` его не было) → loader проставляет дефолт по категории
(`radarForCategory`). `ship_classes.yaml` может задать `radar:` явно (override).

## L1 — персональный радиус обзора

- `domain.Ship.RadarRange float64` (персист `ships.radar_range`, миграция 0046 —
  как другие производные статы: пишется в Create, читается LoadAll; fallback при 0).
- Спавнеры (`spawnStarter`, `buildWarship`, NPC, quest) проставляют
  `RadarRange = cls.Radar`.
- `broadcastPatches`: `sub.Radius` каждый тик берётся из `RadarRange` корабля
  владельца подписки (fallback `cfg.AOIRadius`). Малый радар фильтрует
  корабли/ракеты/дроны/контейнеры/лазеры (всё, что уже фильтровалось по AOI).
- Фронт: кольцо малого радара вокруг своего корабля (`ownShip.radarRange` в DTO).

## L2 — двухуровневая дальность + дельты статики

- Крупные объекты (станции/верфи/ТС/пирбазы/башни/ворота) видны в
  `RadarRange × K`.
- Статика приходит/уходит **дельтами** per-subscriber по мере движения
  (вместо разовой полной отдачи): подписка трекает видимые `EntityRef`,
  каждый тик считает попавшие в большой радиус → `staticsAdded` / `staticsRemoved`
  в патче. Фронт мёржит дельты в карту статики.
- Второе кольцо `× K` на канвасе.

## L3 — сканер (`up_scanner`)

Ложится на пайплайн эффектов оборудования из 10.14: `RadarRange` добавлен в
`balance.ShipStats`; `baseShipStats` берёт `cls.Radar`; `ApplyEquipmentEffects`
для `up_scanner` множит/прибавляет к радару (по уровню). Установка/снятие
пересчитывают и персистят `radar_range` (как прочие статы), подписка читает
обновлённый `RadarRange` со следующего тика. До 10.14 — n/a (10.14 готов).

Множитель сканера: **+40 % за уровень** (`RadarRange *= 1 + 0.4*level`).

## L4 — стелс (`up_hide`)

- `domain.Ship.IsHidden bool` (RAM-only combat-state, не персистится — как
  `AttackTarget`).
- В `broadcastPatches` скрытый корабль исключается из видимого набора **чужим**
  игрокам, кроме: (а) очень малой дистанции `cfg.StealthDetectRange` (порог
  обнаружения); (б) своих (`ship.PlayerID == sub.PlayerID`); (в) союзников
  (`relations`/клан — не враждебны). Свой корабль виден всегда.
- Активация: MVP — наличие установленного `up_hide` (10.14) включает
  `IsHidden` при спавне/установке; всплытие (`IsHidden=false`) при атаке
  (выстрел лазером/ракетой) — на текущий тик. Энергомодель `always`/всплытие
  при 0 энергии — follow-up.
- Фронт: индикатор «стелс активен» в HUD.

## L5 — спутники (раскрытие сектора)

Объект-спутник вводит 10.15 (`back/docs/specs/satellite.md`). Он же реализовал
базовый reveal — **публичный** (любой живой спутник раскрывал сектор всем
подписчикам). L5 здесь делает reveal **владелец-/клан-гейтированным**, как в
оригинале StarWind («спутники раскрывают видимость… в т.ч. для клана») и как
требует критерий L5 («принадлежащий игроку спутник расширяет ЕГО видимость»).

- **Back:** `broadcastPatches` бустит `sub.Radius` до
  `cfg.SatelliteRevealRadius` только если `satelliteRevealsFor(sub.PlayerID,
  relations)` — у подписчика есть собственный built-спутник в секторе ИЛИ он
  не-враждебен (Friend по `Relations`) владельцу спутника. Враг от чужого
  спутника обзор не получает. `satellitesPresent()` остаётся дешёвым
  once-per-tick гейтом (per-sub проверка запускается только при наличии хотя бы
  одного спутника). Спутник без владельца (`OwnerID == nil`) не раскрывает
  никому. Зеркалит own/ally-логику `hideStealthed` (L4). Стелс применяется
  поверх расширенного окна.
- **Front:** `drawSatelliteCoverage` (`SectorCanvas`) рисует мягкое фиолетовое
  (alpha 0.3, dashed) кольцо зоны покрытия вокруг **своих** built-спутников
  (`ownerID === ownPlayerID`); чужие спутники без кольца. Радиус кольца —
  истинный `SATELLITE_REVEAL_RADIUS` (10000, зеркало бэка). Он ≈2× радиуса
  сектора, поэтому при зуме обзора сектора кольцо целиком уходит за кадр — это
  сознательно (буквальная дальность покрытия, а не заклэмпленная подсказка).

Тесты: `TestUnit_Satellite_RevealGatedByOwner` (owner+ally видят, neutral нет),
`TestUnit_Satellite_RevealsSector` (владелец видит весь сектор).

**Follow-up (вне L5):** приватная зона покрытия с per-clan визуализацией;
прокидка `SatelliteRevealRadius`/`RadarBigMultiplier` в welcome вместо
клиентских констант.
